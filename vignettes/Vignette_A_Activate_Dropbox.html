Vignette A: Activate Dropbox

Introduction

The BIOWELL package includes functions for building a custom survey in Shiny using the BIO-WELL scale and sharing the survey with participants via the internet by deploying them onto the Posit Shiny Server.

In this vignette, we will outline how to register for a Dropbox account and then generate a long-lived Dropbox access token for storing response data from your BIO-WELL survey.

A video walk through of this vignette can be found here on YouTube.

Why is this important?

When your survey is uploaded to the Posit Shiny Server, participant response data cannot be stored locally. Therefore, the BIOWELL package integrates Dropbox for free remote storage of survey response data.

To store data to a Dropbox account over long periods of time, an access token needs to be generated and stored where your Shiny BIO-WELL survey can access it whilst uploaded to the Posit Shiny Server.

To find out more about access tokens for applications, including Dropbox, visit this website.

You only need to do this once when creating your BIO-WELL survey!

Step 1: Installation

Before starting, install the BIOWELL and rdrop2 packages in R. Then load the libraries.

# devtools::install_github("r-a-dobson/BIOWELL") # Install from GitHub
# install.packages("BIOWELL")  # Install from CRAN
# install.packages("rdrop2")
library(BIOWELL)
library(rdrop2)
#> Warning: package 'rdrop2' was built under R version 4.0.5

Step 2: BIO-WELL survey folder

Throughout the BIOWELL framework, your BIO-WELL survey will need to have its own folder. This is where you will save an R script named ‘app.R’ which contains the survey script (build_survey() - Vignette B), as well as the Dropbox access token (Vignette A).

Let’s create a folder to store our BIO-WELL survey and associated data to!

Note the app name that you use here. When creating your Dropbox App folder and your Shiny Server app, it would be useful to keep the name consistent through all three (local, Dropbox and Shiny Server) - but they can be different from each other if you wish.

path_to_folder <- tempdir() # Replace with path to a local folder. e.g. "C:/Users/username/Documents/"

appName <- "my_app_name" # Replace with name for your app. 

dir.create(paste0(path_to_folder,"/",appName))

Step 3: Dropbox account

Next, you need to create a free account on the Dropbox website.

Step 4: Dropbox App

To save your survey data into, you now need to create a Dropbox app within your Dropbox account.

Here you should:

  • Select “scoped access”

  • Select “app folder”

  • Choose a name for your app

Create Dropbox App

Step 5: Dropbox App set-up

Now you can access your Dropbox App Console.

Scroll down to OAUTH2: Redirect URIs and add “http://localhost:1410/”.

Add redirect URI

Then, go to the Permissions tab and tick the relevant boxes. You need to give the app permission to read and write files.

Add Dropbox App permissions

Return back to the app’s Settings tab, and scroll down to reveal your app’s key and secret. This is required for activating the access token! Remember to never share your app’s key and secret.

Get Dropbox name, key and secret

Let’s save our Dropbox App details in the R environment.

Dropbox_App_Name <- "my_app_name" # Replace with your Dropbox App name. 

Dropbox_App_Key <- "string-of-letters-and-numbers" # Replace with your Dropbox App key.  

Dropbox_App_Secret <- "string-of-letters-and-numbers" # Replace with your Dropbox App secret.

Step 6: Activate Dropbox access token

Now you have all of the details needed to generate the Dropbox access token. Run the code below. This will open a window in your browser to authenticate the token.

Remember to provide the function with the path to the BIO-WELL survey folder (BW_app_path) that you created in Step 1 of the vignette.

activate_dropbox(Dropbox_App_Name = Dropbox_App_Name,
                 Dropbox_App_Key = Dropbox_App_Key,
                 Dropbox_App_Secret = Dropbox_App_Secret,
                 BW_app_path = paste0(path_to_folder,"/",appName))

Once you have authenticated the token, the token will be saved in a “.secrets” folder within your BIO-WELL survey directory.

When you upload your BIO-WELL survey to the Shiny servers, the Shiny App will now be able to remotely store survey responses to your Dropbox App folder. These data can be retrieved using the BIOWELL package’s download_data() function.
